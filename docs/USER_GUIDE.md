# Пользовательская документация

## Быстрый старт
### Windows
1. Перейди в `MSHP-IDE-Windows/`.
2. Запусти `МШПаха.Оффлайн.bat`.

### macOS / Linux
1. Перейди в `MSHP-IDE-macOS/` или `MSHP-IDE-Linux/`.
2. В папке уже лежит portable Python **с Tkinter/Tk** для двух архитектур:
   - `python/x86_64` (Intel)
   - `python/aarch64` (Apple Silicon / ARM)
3. Выполни один раз: `chmod +x МШПаха.Оффлайн.sh`.
4. Запусти: `./МШПаха.Оффлайн.sh`.

## Интерфейс
- **Вкладки** — работа с модулями (файлами).
- **Кнопки запуска** — отдельная панель: зелёный «Запустить main.py» и красный «Остановить».
- **Остановить** неактивна (бледная), когда код не запущен.
- **Поле ввода** — между редактором и консолью.
- **Консоль** — вывод программы, ошибки и статусы.
- **Тёмная тема** — переключатель справа.
- **Turtle‑панель** — появляется справа, если в коде есть `import turtle`.
- **Полезное** — меню с быстрыми действиями (например «Сделать отступ»).

## Настройки
Открываются через меню **«⚙ Настройки»**.
- Тёмная тема.
- Сохранение перед запуском: спрашивать/всегда/никогда.

## Работа с файлами
- **Новый / Открыть / Сохранить / Закрыть** — на верхней панели.
- При закрытии приложения всегда будет вопрос о сохранении несохранённых вкладок.
- Первый новый модуль называется `main.py`.
- Все последующие — `module1.py`, `module2.py` и т.д.
- Если `main.py` пустой, «Открыть» загрузит файл прямо в него.
- Нумерация новых модулей берёт **минимальный свободный номер**.

## Редактирование
- **Tab** делает отступ для всех выделенных строк.
- **Полезное → Сделать отступ** — вручную добавляет отступ слева у выделения.

## Импорт без сохранения
- При запуске несохранённого файла IDE создаёт временную «виртуальную папку» в `.runtime/`.
- В неё попадают **все открытые вкладки** (включая несохранённые).
- Поэтому `import module1` / `import module2` и импорты между вкладками работают без ручного сохранения.

## Запуск и ввод
- Нажми **F5** или кнопку «Запустить main.py».
- При первом запуске IDE спросит, сохранять ли файл перед запуском:
  - **Да** → будет сохранять всегда.
  - **Нет** → будет запускать временный файл (в `.runtime/`).
- Ввод:
  - **Enter** — отправить в программу.
  - **Ctrl+Enter** — новая строка в поле ввода.
- Каждый ввод отображается в консоли с префиксом `>`.
- Когда программа ждёт ввод, фокус автоматически переходит в поле ввода и подсвечивается.
- В turtle‑режиме `input()` работает через поле ввода IDE (без консольного окна).

## Turtle
- Если в коде есть `import turtle`, справа появляется встроенная панель.
- Рисунок остаётся на экране после завершения выполнения (не исчезает).
- Отдельное всплывающее окно не появляется.
- Координаты соответствуют реальному размеру панели, центр — в точке (0, 0).
- Встроенный режим игнорирует `screen.mainloop()` — приложение продолжает работать.
- Фокус автоматически переводится в панель turtle, чтобы работали клавиши управления.

## Что включено
- Полный **стандартный Python** (stdlib) + Tkinter.
- Встроенных внешних библиотек нет — это сделано ради минимального размера.

## Как добавить свои библиотеки
1. Windows: создай каталог `python/Lib/site-packages` (если его нет).
2. macOS/Linux: создай каталог `python/<архитектура>/lib/python3.13/site-packages`.
3. Скопируй туда библиотеку (папку или `.whl`, распакованный).

## Ограничения
- Одновременно работает только один запущенный процесс.
- Встроенного `pip` нет (его удалили ради размера).
- Терминал‑виджета нет — используется обычный ввод/вывод.

## Экономия места
- Если тебе нужна только одна архитектура, можно удалить вторую папку:
  `python/x86_64` или `python/aarch64`.
